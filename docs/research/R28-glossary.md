## Key concept definitions

**Vintage / Vintage-correct**
A *vintage* is the value of a time series as published at a specific date in history, before any subsequent revisions. *Vintage-correct backtesting* uses each forecast's historical input vintage rather than the latest revised data — the only kind of backtest that simulates what the forecaster actually saw.

**Tier-V vs. Tier-T**
**Tier-V (vintage-eligible)** countries have publicly-archived real-time vintages and appear on the OPENGEM public leaderboard. **Tier-T (tracked-only)** countries appear in the dashboard with latest-revised values but are explicitly excluded from leaderboard claims.

**The hash quintuple**
`(code_sha, vintage_hash, prior_hash, posterior_hash, run_id)` — the provenance triplet that makes every OPENGEM forecast bit-identically reproducible.

**The V&V matrix**
17 cells at IOC, ~25 at FOC, each specifying (variable, horizon, country-set, metric, benchmark, threshold). Replaces a single "beat AR(1) on X%" gate from rev B.

**The wider information surface**
Per R06: four domains (Markets, Supply Chain, Geopolitics, Media) ingested as L3 covariates and exposed via Situation Subsystem endpoints.

**The L3 workhorse layer**
Per R03 / R14: the baseline forecast producer is L3 only (DFM + ML residual + large BVAR variants combined by BMA). L1 (US-only) and L2 (annual cadence) are narrative/scenario satellites.

**The FRED-substitution policy**
Per R05 / R09: OPENGEM does not cache or train on FRED Content. Upstream US agencies (BEA / BLS / FRB / Treasury / Census) are the persistent source of truth for US data. FRED is permitted only for series discovery, not persistence.
